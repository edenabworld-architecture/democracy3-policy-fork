# Democracy 3.0 국가 시범검토 10건 운영·검증 프로토콜

**버전:** 0.13  
**창안 및 원안 설계:** 박찬성  
**적용 범위:** 대한민국 국회 법안 국가 시범검토 10건

## 1. 목적

Democracy 3.0 정책포크는 법안을 단순히 요약하거나 찬반투표에 부치는 데 그치지 않는다. 법안의 문제진단, 주장, 근거, 권한이동, 비용, 악용경로, 방어장치와 대체조문을 공개적으로 분해하고 검증한다.

국가 시범검토 10건은 최신 법안 전체를 자동 감시하는 넓은 수집층과, 제한된 수의 고영향 법안을 실제 조문·근거 수준으로 깊게 검토하는 집중검증층을 연결하는 첫 운영단위다.

## 2. 시범대상 선정과 고정 원칙

최초 대상은 다음 기준을 합산하여 선정한다.

- 기본권과 민주적 대표성에 미치는 영향
- 권한의 집중·이동·견제구조 변화
- 재정·세수·민간 준수비용
- 규제·제재·행정재량
- 수혜자와 비용부담자의 범위
- 악용·우회·기관포획 가능성
- 공식자료 접근 가능성
- 교육·보건·노동·주거·디지털·재정 등 분야 다양성

한 번 확정된 10건은 새로운 법안이 추가되더라도 자동으로 교체하지 않는다. 고정된 법안이 현재 수집창에서 사라진 경우에만 대체후보를 넣으며, 교체 사실과 사유를 공개한다.

자동추천 후보와 최종 시범대상은 구분한다. 장기적으로는 운영자 박찬성의 검토를 거친 뒤 최종 확정 상태를 기록한다.

## 3. 공식문서 인정 기준

다음 조건 중 하나 이상을 충족해야 해당 법안의 공식문서로 집계한다.

- 문서 URL이나 파라미터에 법안 ID 또는 의안번호가 포함됨
- PDF·HWP·HWPX·DOCX·XLSX 등 실제 파일 주소임
- 첨부파일 고유번호나 다운로드 식별자가 존재함
- 문서 제목에 법안명·의안번호·조문대비표·비용추계서 등이 명시됨
- 국회 또는 소관기관의 법안별 첨부파일 다운로드 경로임

회의록 홈페이지, 의안정보 홈페이지, 법령정보 홈페이지 등 공통 메뉴는 공식문서 수에서 제외하고 `관련 시스템 링크`로 별도 표시한다.

문서의 위치를 발견했다는 사실은 내용을 검토했다는 의미가 아니다. 다음 상태를 분리한다.

1. 위치 확인 필요
2. 법안별 문서 위치 발견
3. 파일 확보
4. 본문 추출
5. 조문·근거 대조
6. 사람 검토 완료

## 4. 성숙도 점수

단일 진행률은 사용하지 않는다. 서로 다른 네 종류의 성숙도를 분리한다.

### 4.1 구조화 준비도

다음 검토 틀이 생성됐는지를 평가한다.

- 공식 제안이유 및 주요내용
- 위험프로필
- 주장 원장
- 레드팀 공격경로
- A/B/C 포크 초안

구조화 준비도가 높아도 증거검증이 끝난 것은 아니다.

### 4.2 공식자료 확보도

다음 자료의 실제 확보·연결 정도를 평가한다.

- 국회 상세페이지
- 조문 참조
- 법률안 조문 전문
- 신·구조문 대비표
- 비용추계서 또는 미첨부 사유
- 위원회 검토·심사보고서
- 법안별 회의록·토론자료

### 4.3 독립근거 검증도

발의자의 주장마다 독립적인 통계·연구·감사·판례·공식 운영자료를 연결하고, 입증·부분입증·기각·판정유보 상태를 기록한다.

### 4.4 사람검토

자동분석과 자료연결을 운영자 또는 검토자가 실제로 읽고 확정했는지를 기록한다.

- 운영자 검토 전
- 운영자 검토 중
- 박찬성 운영자 검토 완료
- 외부 전문가 검토 중
- 외부 전문가 검토 완료
- 반론 접수 후 재검토

## 5. 주장 유형과 증거기준

발의자의 문장은 다음과 같이 분류한다.

- 법적·헌법적 주장
- 국제·제도 비교 주장
- 재정·비용 주장
- 기본권·권리 영향 주장
- 수치·규모 주장
- 인과·원인 주장
- 집행·행정역량 주장
- 효과·편익 주장
- 규범·당위 주장
- 문제 진단

주장 유형마다 필요한 증거와 기각조건을 다르게 설정한다.

예를 들어 헌법적 주장은 통계의 분모가 아니라 헌법 조문, 판례와 법리를 검토해야 한다. 국제비교 주장은 일부 국가의 이름만 나열해서는 충분하지 않으며, 국가별 실제 법령과 예외, 지방자치구조와 정치제도의 차이를 함께 검토해야 한다.

## 6. 조문 대조 원칙

조문 검토는 다음 순서로 진행한다.

1. 공식 요약에 언급된 조문번호 추출
2. 법률안 전문 확보
3. 현행법 원문 확보
4. 신·구조문 문장 단위 비교
5. 권리·의무·권한·예외·위임·제재 변화 표시
6. 시행령·규칙 위임 범위 검토
7. 포크 A/B/C의 실제 대체조문 작성
8. 각 포크의 방어효과와 잔여위험 비교

같은 조문을 `안 제21조`와 `제21조`로 중복 집계하지 않는다. 조·항·호를 정규화하여 하나의 조문 참조로 관리한다.

## 7. 레드팀과 포크 연결

각 공격경로는 다음 요소를 포함한다.

- 공격 또는 우회 주체
- 동기와 획득이익
- 악용·회피 경로
- 조기경보 지표
- 방어장치
- 방어 후 남는 위험

A/B/C 포크는 단순 아이디어 목록이 아니라 어떤 공격을 차단하는지, 어떤 위험을 새로 만드는지, 어느 조문을 어떻게 바꾸는지 연결해야 한다.

## 8. 완료 단계

- **S1 구조화·자료확보 준비:** 주장·위험·포크 검토 틀이 생성됨
- **S2 공식문서 대조 중:** 법률안·대비표·비용자료 등 공식문서가 확보됨
- **S3 독립근거 검증 중:** 핵심 주장에 외부 근거와 반증이 연결됨
- **S4 운영자 검토 완료:** 박찬성 운영자 검토와 수정이력 기록
- **S5 전문가 검증 완료:** 법제·분야 전문가의 검토와 반론처리 완료

## 9. 공개성과 정정

- 공식자료, 자동추론, 사람검토와 시민의견을 구분한다.
- 자동분석을 사실 또는 법률자문으로 표시하지 않는다.
- 오류는 조용히 덮지 않고 변경이력을 남긴다.
- 반론은 삭제 대상이 아니라 검증과 수정의 입력으로 보존한다.
- 분석규칙과 점수산식의 버전을 공개한다.
- 특정 정당·진영·발의자를 일관되게 유리하거나 불리하게 처리하는지 편향감사를 실시한다.

## 10. 국가 시범사업의 최소 성과물

각 대상 법안마다 다음을 남기는 것을 목표로 한다.

- 국민용 1분 설명
- 공식 제안이유와 조문 원문
- 현행법·개정안 비교
- 핵심 주장과 독립근거
- 비용·권한·기본권 영향
- 강화된 레드팀 검토
- 실제 A/B/C 대체조문
- 운영자·전문가 검토기록
- 반론과 수정이력
- 통과 후 시행·성과·부작용 추적설계

이 프로토콜은 국가 시범검토 과정에서 발견된 오류와 반론에 따라 계속 개정한다.


## 11. 검증된 인간 작업의 영속성

자동수집 결과는 매 실행마다 다시 생성되지만, 운영자와 전문가가 확인한 결과는 `review_overrides.json`에 별도로 보존한다.

- 자동생성 파일을 직접 수정하지 않는다.
- 공식문서·독립근거·조문대조·실제 포크·검토·정정은 수동검증 원장에 기록한다.
- 자동실행은 수동검증 원장을 검증한 후 새 데이터에 병합한다.
- 출처가 없거나 검토자 기록이 부족한 완료표시는 공개 전 검사에서 차단한다.
- 고정 시범법안이 최신 100건 창 밖으로 이동해도 직전 정상 데이터에서 보존한다.

## 12. 공개 전 배포검사

`validate_release.py`는 다음을 확인한다.

- manifest의 10건과 공개 시범보고서의 ID·순서 일치
- 공통 내비게이션 링크의 공식문서 오인 방지
- 근거 없는 입증·반증 판정 방지
- 출처 없는 현행법·개정안 조문 방지
- 실제 조문이 없는 조문단계 포크 방지
- 검토자·일자 없는 사람검토 완료 방지
- 전문가 소속·전문분야 없는 전문가 완료 방지
- 경량 목록과 개별 보고서의 누락 방지

검사 실패 시 새 데이터는 커밋하지 않는다.
